# 公众号写作员工包

一套标准化的 AI 员工打包规范，把公众号写作的完整工作流拆成五个核心员工 + 一个可选员工，串行协作。

## 设计原则

本员工包严格遵循以下设计规范：

1. **IDENTITY.md / SOUL.md 只负责软约束** — 定义员工是谁、怎么说话、有什么价值观、偏好什么输出风格。不承担安全边界
2. **policy.yaml 负责硬边界** — 工具白名单、文件访问范围、网络访问、脚本执行权限。由 adapter 编译成各运行时真实支持的权限配置
3. **USER.md 不进入公开包** — 用户画像、工作空间信息、私密偏好、密钥在安装后生成本地私有配置层。模板见 `私有配置模板/`
4. **Skill 按最强 runtime 设计** — canonical 格式对齐 Claude Code 的 SKILL.md 思路（frontmatter、when_to_use、依赖、工具需求、references、scripts、assets），再按 `adapters/` 配置向 Hermes / Codex 做能力协商
5. **Adapter 是能力协商器** — 不是格式转换器。`adapters/` 目录下三个 yaml 声明各 runtime 的能力矩阵和降级策略

## 员工

| 员工 | ID | 负责步骤 | 一句话定位 | 可选 |
|------|----|---------|-----------|:---:|
| 账号配置员 | `account-configurator` | Step 0 | 生成本地私有配置，配置账号、风格卡与可选发布接入 | |
| 素材捕手 | `content-scout` | Step 1-4 | 锁定风格、读源素材、搜索补充、搭框架 | |
| 事实核查员 | `fact-checker` | Step 5, 8 | 数据核查建表 + 写完逐句反查来源 | |
| 写手 | `writer` | Step 6-7 | 拿核查通过的数据和框架，按模板写 HTML + 出标题 | |
| 审校 | `editor` | Step 9-11 | 降 AI、9 项自检、日期核实、归档 | |
| 发布员 | `publisher` | 发布 | 将归档的 HTML 推送到平台草稿箱（需安装后配置） | ✅ |

Agent 加载本包后必须先启动「账号配置员」。账号配置员先生成本地私有配置，确认公众号账号、工作区和风格卡；如果用户需要自动推送草稿，再引导用户去微信开发者平台获取 AppID 和 AppSecret，并协助查询本机公网出口 IP，用于配置平台网络允许列表。

## 协作流程

```
账号配置员 ──→ 素材捕手 ──→ 事实核查员 ──→ 写手 ──→ 审校 ──→ 发布员（可选）
 (账号+配置)    (搜+搭)      (核数据)      (写+标)    (降+检+存)   (推草稿)
```

每个员工有硬门禁：上一个不交付，下一个不启动。

## 首次启动

第一次交付给用户时，建议先阅读并执行：[首次运行自检与故障反馈.md](./首次运行自检与故障反馈.md)。

首次加载本包时，Agent 应按以下顺序处理：

1. 检查本地私有配置层是否存在
2. 引导用户基于 `私有配置模板/README.md` 生成 `USER.md`
3. 填写公众号账号、工作区、风格卡
4. 询问是否启用发布员
5. 如启用发布员，引导用户获取 AppID、AppSecret，查询本机出口 IP，并完成平台网络允许列表配置
6. 配置完成或用户选择跳过发布后，进入素材捕手

## 写作风格卡

`共享资源/style.md` 提供了一份参考风格（适配科技/财经类公众号，说话犀利、短句、有网感）。

**这不是唯一风格。** 用户可以：
1. 直接用这份风格
2. 复制一份改成自己的 → 在 manifest.yaml 里指向新文件
3. 按 `共享资源/style-schema.md` 的字段定义写一份全新的

## 安装

推荐安装完整总包：

```bash
npx skills add https://github.com/Shinchan-crayon/wechat-writing-employee-pack
```

安装后由 adapter 读取 `manifest.yaml`、各员工 `policy.yaml`、`skills/` 和 `adapters/` 完成加载。

安装后：
1. 复制 `私有配置模板/` 为本地私有配置目录
2. 按 `私有配置模板/README.md` 在本地生成并填写 `USER.md`
3. 首次运行时由账号配置员引导完成账号、风格卡和可选发布接入配置
4. 如无法正常启动，按 `首次运行自检与故障反馈.md` 收集问题信息再反馈

## ThinkAI 上架入口说明

本仓库采用“一个仓库，多入口上架”的方式：

- **推荐入口：公众号写作员工包** — 安装完整总包，按工作流顺序运行
- **子入口：账号配置员** — 初始化账号、风格卡、本地私有配置和可选发布接入
- **子入口：素材捕手** — 读取风格和源素材，搜索补充信息，搭建文章框架
- **子入口：事实核查员** — 对数据、价格、案例、引用进行写前和写后核查
- **子入口：写手** — 基于已核查框架生成公众号 HTML 和标题候选
- **子入口：审校** — 降 AI 味、做 9 项自检、核实日期并归档
- **子入口：发布员** — 可选，把归档 HTML 推到平台草稿箱，不最终发布

子入口用于 ThinkAI Skill 广场展示和说明，不建议脱离总包单独运行。6 个员工依赖总包的共享资源、模板、私有配置层和上游交付物。

首次运行必须先启动「账号配置员」。`USER.md`、`.env`、真实 AppSecret、真实 access_token 和其他私有凭证永不进入公开仓库。发布员默认可选；没有完成平台凭证和网络允许列表配置前，只运行写作闭环。

## 兼容性

| Runtime | 状态 | 说明 |
|---------|------|------|
| Claude Code | ✅ 全功能 | 全量加载 identity/soul/policy/skills |
| Hermes | ✅ 目标运行时 | 支持完整写作流程；浏览器搜索和发布引导依赖本地浏览器连接、登录状态和网络可用性 |
| Codex | ⚠️ 部分降级 | browser 受限，MCP 不可用，gates 改为用户提示 |

## 能力边界

- 面向中文科技/财经/行业分析类公众号
- 输出格式：HTML 文件（可直接粘贴到公众号编辑器）
- 默认无配图，用户要求时才生成
- 可选发布功能由账号配置员在首次运行时引导配置，只推送到草稿箱，不替用户最终发布
