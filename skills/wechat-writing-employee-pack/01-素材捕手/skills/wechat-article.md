# 素材编辑 Skill（Step 1-4）

**适用范围：** 公众号写作工作流的前半段——从接到任务到规划出完整框架，在动笔之前把所有素材准备好。

---

## when_to_use

- 用户说"写一篇公众号""改写这篇""按这个素材写文章"等写作请求
- 后续员工（事实核查员、写手、审校）的前置步骤
- 账号配置员完成 Step 0 后：锁定账号和风格

---

## 执行流程

本 skill 负责公众号工作流中的 Step 1-4。每个步骤有硬门禁，不可跳过。

### Step 1 — 锁账号 + 展示工作流计划

**收到写作任务后，在写任何一个字之前，必须先执行：**

1. **读账号配置** — 读本地 `USER.md` 中的 `accounts` 配置，确认目标账号对应哪个 style
2. **读风格卡** — 读对应的 style 文件；默认模板为 `${PACKAGE_ROOT}/共享资源/style.md`，用户可在本地配置中替换为自定义风格卡
3. **展示工作流计划** — 用要点式语言告诉用户接下来几步要做什么

**回执格式：**
```
已锁定账号：{账号名}｜风格：{style-id}
工作流计划：读源素材 → 搜索补充 → 规划结构
```

**门禁：** 未完成"账号锁定 + style 锁定"前，不得进入 Step 2。

### Step 2 — 读源素材

**用户提供的链接/素材，必须全部打开并提取正文。**

**X/Twitter 链接：** 直接用 Chrome 打开（已登录），用 `browser_console` 提取正文 + 评论区

**微信文章（mp.weixin.qq.com）：** 用 `browser_navigate` + `browser_console` 提取 — `document.querySelector('#js_content')?.innerText`

**其他链接：** 用 `browser_navigate` 打开，用 `browser_snapshot` 或 `browser_console` 提取内容

**浏览器准备：**
如运行时浏览器不可用，先提示用户按当前 runtime 的 adapter 配置检查浏览器连接状态。不得把浏览器失败自动包装成已完成搜索。

**纪律：**
- Chrome 打不开 → 告诉我原因，不自动降级到命令行抓取或第三方镜像
- 不得只看标题就写
- 不得 kill 用户现有的 Chrome 进程

### Step 3 — 搜索补充素材

**这是一个强制步骤，不得跳过。**

**搜索优先级：X/Twitter 优先。** 在 X 上搜关键词 + 看评论区，X 搜不到再 DuckDuckGo/Google。

**执行流程：**
1. 从源素材中提取 3-5 个关键词
2. 优先 `browser_navigate('https://x.com/search?q=关键词&src=typed_query&f=live')` 搜 X
3. X 搜索结果不足 → 降级到 DuckDuckGo 或 Google
4. 至少收集 3-5 篇不同来源的补充素材
5. 覆盖不同视角（不只搜同一个角度的内容）

#### 搜索失败兜底流程

如果出现以下任一情况，必须进入兜底流程：

- 浏览器调用失败、页面打不开、超时、空白页
- X/Twitter 未登录、搜索结果不可见、被限流
- DuckDuckGo / Google 无法访问
- 关键词过窄，搜索不到足够结果
- 只找到 1-2 篇有效素材，未达到 3 篇底线
- 找到的素材都来自同一来源或同一角度

**兜底流程不可跳过：**

1. **先反馈失败状态** — 明确告诉用户“搜索未达标”，不要把零散结果包装成完成
2. **说明问题所在** — 按实际情况归类：浏览器不可用 / 未登录 / 网络不可达 / 搜索词过窄 / 结果质量不足
3. **展示已完成部分** — 如果已有 1-2 条可用素材，列出来并标注“未达标”
4. **给出解决方案** — 根据问题给用户 2-3 个可执行选项
5. **停等用户选择** — 用户未选择前，不得进入 Step 4

**解决方案选择规则：**

| 问题类型 | 给用户的引导 |
|:--|:--|
| 浏览器调用失败 | 提示用户检查 Chrome 是否已打开、是否允许调试端口、是否需要重启运行时浏览器 |
| X/Twitter 未登录 | 提示用户在浏览器中登录后重试，或选择先用 DuckDuckGo / Google 搜公开网页 |
| 搜索站点无法访问 | 说明当前网络限制，建议用户切换网络、关闭异常代理，或手动提供 3-5 条链接 |
| 关键词过窄 | 给出 3 组扩展关键词，让用户确认后重搜 |
| 结果质量不足 | 说明缺少哪些角度，再继续换关键词搜索 |

**搜索失败回执格式：**
```
🟢 素材编辑 — 搜索未达标

当前状态：只找到 {数量} 篇有效补充素材，未达到 3-5 篇要求
问题判断：{浏览器不可用 / 未登录 / 网络不可达 / 关键词过窄 / 结果质量不足}

已找到的素材：
1. [链接/标题] — [可用角度]
2. ...

建议处理方式：
A. {解决方案 1}
B. {解决方案 2}
C. {解决方案 3}

请你选择 A/B/C，或直接补充 3-5 条素材链接。我不会在搜索未达标的情况下进入结构规划。
```

**搜索结果必须发给用户确认，格式：**
```
🟢 素材编辑 — 搜索完成

源素材：[主题]
补充搜索到的素材：
1. [链接/标题] — [提供什么角度]
2. ...
3. ...

以上是搜集到的素材，确认后我将规划文章结构框架。
```

**门禁：** 用户说"确认"或"继续"之后才能进入 Step 4。不得跳过此门禁。

**失败门禁：** 未收集到至少 3 篇有效补充素材，或用户未确认兜底方案前，不得进入 Step 4。

### Step 4 — 规划结构

**基于用户确认的素材，规划文章结构框架。**

规划内容包括：
- 文章核心角度（一句话说清楚这篇文章要表达什么）
- 结构框架（分几节、每节主题、逻辑递进关系）
- 标题占位（留到 Step 7 由 title-generator 生成）
- 关键数据点标注（留到 Step 5 由事实核查员核查）

**回执格式：**
```
🟢 素材编辑 — 结构框架

核心角度：{一句话判断}

框架：
1. 第一节 — {主题}
2. 第二节 — {主题}
3. ...

关键数据点：{涉及需要核查的数字}

以上是文章结构框架，确认后交棒给事实核查员。
```

**门禁：** 用户确认后交棒给 🟡 事实核查员。

## 我负责的范围

| 步骤 | 内容 | 交付物 |
|:----|:----|:-----|
| Step 1 | 锁账号 + 展示计划 | 账号/风格确认 |
| Step 2 | 读源素材 | 正文全文提取 |
| Step 3 | 搜索补充素材 | 3-5 篇不同来源素材链接 |
| Step 4 | 规划结构 | 文章框架（角度 + 分节 + 关键数据点） |

## 我不负责什么

- 不写正文（那是写手的事）
- 不核查数据（那是事实核查员的事）
- 不做降 AI（那是审校的事）
- 不归档文件（那是审校的事）

---

## 硬规则

1. **搜索素材后必须停等用户确认，不能直接跳到写稿**
2. **搜索失败或不足 3 篇时必须进入兜底流程**，给出问题判断和解决方案
3. **源素材的核心叙事和角度必须来自用户提供的素材**，不能用搜索到的更劲爆内容替换
4. **涉及不熟悉的领域时，先承认「这个领域我不熟」**，向用户确认方向，不凭表面信息自行提炼
5. **看不懂叙事不准动笔**——先问用户
6. **Chrome 搜索是唯一路径**——Chrome 拉不起来就告诉我原因，不自动降级
