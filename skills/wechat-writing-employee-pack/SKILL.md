# 公众号写作员工包

这是一个公众号写作工作流总包。不要把它当成“一句话生成全文”的写作提示词使用。

核心原则：先配置账号和风格，再整理素材和核查事实，最后写正文、审校归档。发布员是可选能力，只创建公众号草稿，不做最终发布。

## 默认流程

按下面顺序执行：

```text
账号配置员 -> 素材捕手 -> 事实核查员 -> 写手 -> 审校 -> 发布员（可选）
```

对应目录：

- `00-账号配置员/`：初始化账号、工作区、写作风格、本地私有配置和可选发布接入
- `01-素材捕手/`：读取素材、补充搜索、整理来源、搭建文章框架
- `02-事实核查员/`：核查数据、价格、案例、引用和时间线
- `03-写手/`：基于已核查框架生成公众号 HTML 正文和标题候选
- `04-审校/`：降 AI 味、做自检、核实日期并归档
- `05-发布员/`：可选，把归档 HTML 推到公众号草稿箱，不最终发布

## 必须遵守

- 第一次使用必须先执行 `00-账号配置员/`。
- 不要跳过事实核查直接写正文，除非用户明确要求快速草稿；即使快速草稿，也要标出事实风险。
- `USER.md`、`.env`、真实 AppSecret、真实 access_token 只允许保留在用户本地，不得写入公开仓库或最终交付内容。
- 发布员默认关闭。只有用户明确要求创建公众号草稿，并且本地凭证已配置完成时，才进入发布员。
- 发布员只创建草稿箱草稿，不执行最终群发或发布。

## 使用入口

如果用户是第一次使用，先读取：

- `README.md`
- `首次运行自检与故障反馈.md`
- `用户配置模板/README.md`
- `00-账号配置员/manifest.yaml`

如果用户已经配置过账号和风格，再按任务需要读取对应员工目录的 `manifest.yaml`、`IDENTITY.md`、`SOUL.md`、`policy.yaml` 和 `skills/` 文件。

## 常用任务路由

- 用户要配置账号、风格、工作区：进入 `00-账号配置员/`
- 用户给选题、链接、素材，希望整理文章方向：进入 `01-素材捕手/`
- 用户要检查数据、价格、引用、案例、时间线：进入 `02-事实核查员/`
- 用户要生成公众号正文、HTML、标题候选：确认事实核查完成后进入 `03-写手/`
- 用户要润色、降 AI 味、终稿检查、归档：进入 `04-审校/`
- 用户明确要推送到公众号草稿箱：确认本地凭证后进入 `05-发布员/`
