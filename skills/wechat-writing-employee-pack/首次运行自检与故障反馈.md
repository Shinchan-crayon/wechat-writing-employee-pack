# 首次运行自检与故障反馈

这份说明给第一次拿到本包的用户使用。不要一上来直接让写手写文章，先按下面顺序跑通最小闭环。

## 一、先确认运行方式

请先确认你使用的是哪种运行时：

- Claude Code：优先支持完整能力
- Codex：可用，但部分能力会降级
- Hermes：目标运行时，支持完整写作流程；浏览器搜索和发布引导依赖本地浏览器连接、登录状态和网络可用性

如果你使用 Hermes，就优先按 Hermes 目标运行时启动；切换到 Codex 时，再按降级模式处理。

## 二、首次运行顺序

把包交给 Agent 后，先对 Agent 说：

```text
请读取这个员工包，先执行账号配置员，不要跳到写手。
```

Agent 应按这个顺序启动：

```text
账号配置员 → 素材捕手 → 事实核查员 → 写手 → 审校 → 发布员（可选）
```

发布员默认不要启用。先跑通写作闭环，再配置自动推送草稿。

## 三、账号配置员必须先完成

账号配置员需要先确认：

1. 用户本地配置是否存在
2. 是否已经基于 `用户配置模板/README.md` 生成本地 `USER.md`
3. `USER.md` 里是否填写工作区、公众号账号名、风格卡路径
4. 是否启用发布员

真实 `USER.md` 只放在用户本地，不放进公开包。

## 四、先跑纯写作闭环

第一次试用建议只跑核心写作流程：

```text
账号配置员 → 素材捕手 → 事实核查员 → 写手 → 审校
```

先不要启用发布员，也不要配置 AppID、AppSecret。

最小测试任务可以这样说：

```text
请用这个包跑一篇测试公众号文章。先执行账号配置员，然后进入素材捕手。发布员先跳过。
```

## 五、素材搜集失败时怎么处理

素材捕手如果没有搜到 3-5 篇有效补充素材，不允许直接跳到写手。

它应该返回：

1. 当前搜集到了几篇
2. 为什么未达标
3. 是浏览器不可用、未登录、网络不可达、关键词过窄，还是结果质量不足
4. 给出 2-3 个解决方案
5. 等用户选择后再继续

如果 Agent 直接跳到写稿，说明它没有遵守素材捕手的门禁。

## 六、发布员什么时候启用

只有核心写作闭环跑通后，才建议启用发布员。

启用发布员前需要准备：

1. 已认证的平台账号
2. AppID
3. AppSecret
4. 本机公网出口 IP
5. 平台网络允许列表
6. 本地 `USER.md` 中的发布员配置

发布员只负责推送到草稿箱，不替用户最终发布。

## 七、常见问题判断

| 问题 | 常见原因 | 处理方式 |
|---|---|---|
| Agent 直接找写手 | 没按首次运行顺序启动 | 重新要求先执行账号配置员 |
| 找不到 `USER.md` | 用户本地配置未生成 | 按 `用户配置模板/README.md` 生成 |
| 找不到模板或脚本 | 包路径没有被 adapter 正确解析 | 检查 `${PACKAGE_ROOT}` 是否指向本包根目录 |
| 素材搜索失败 | 浏览器、登录、网络或关键词问题 | 按素材捕手兜底流程处理 |
| 写手无法生成 HTML | 模板路径或工作区路径配置错误 | 检查 `模板/article.html` 和 `workspace.base` |
| verify 不通过 | 文章字数、摘要、句长、标红或段落不达标 | 按脚本提示修正文稿 |
| 发布员不能用 | 平台凭证或网络权限未配置 | 回到账号配置员完成发布接入 |

## 八、反馈问题时请带这些信息

如果无法正常使用，请把下面信息发给维护者：

```text
1. 使用的运行时：Codex / Claude Code / Hermes
2. 当前包路径：
3. 卡在哪个员工：账号配置员 / 素材捕手 / 事实核查员 / 写手 / 审校 / 发布员
4. 卡在哪一步：
5. 报错原文：
6. 是否已经生成本地 USER.md：
7. 是否启用发布员：
8. 是否能读取 模板/article.html：
9. 是否能运行 脚本/verify-article.py：
10. 你希望只跑写作闭环，还是也要推送草稿：
```

没有这些信息，维护者很难判断是包结构问题、用户配置问题，还是运行时能力限制。

## 九、交付边界

本包当前交付的是“公众号写作员工工作流模板”。

它保证：

- 员工职责清楚
- 工作流顺序清楚
- 核心写作闭环完整
- 用户配置不进入公开包
- 发布员是可选能力

它不保证：

- 未完成本地浏览器连接、登录或网络准备时仍能执行浏览器搜索和发布引导
- 用户未配置 `USER.md` 时也能直接跑
- 未认证平台账号也能调用发布接口
- 自动替用户最终发布文章
