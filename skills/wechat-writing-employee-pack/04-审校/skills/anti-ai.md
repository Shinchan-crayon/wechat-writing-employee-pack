# 降 AI Skill

**适用范围：** 文章写完后、归档前的降 AI 处理。在词层面工作，不动 HTML 格式结构。

---

## when_to_use

- 事实核查员（🟡）完成 Step 8 并交棒后
- 用户确认"开始降 AI"后
- 仅做语言层面轻量替换，不破坏 HTML 骨架

---

## 五条核心降 AI 原则

所有操作围绕这五条展开。**每轮降 AI 后自检这五条是否都触及了。**

1. **删除填充短语** — 去除开场白、强调性拐杖词、无信息量的过渡句。直接给事实
2. **打破公式结构** — 避免二元对比（不是…而是…）、戏剧性分段、修辞性设置
3. **变化节奏** — 混合句子长度。两项优于三项。段落结尾要多样化
4. **信任读者** — 直接陈述事实。跳过软化、辩解、手把手引导。不替读者总结
5. **删除金句** — 如果一句话听起来像可引用的名言或朋友圈签名，重写它

1. 收到 🟡 事实核查员交棒（核查全绿）

### 降 AI 操作

#### 正确姿势

- ✅ 只在**语言层面**工作：换高频词、拆长句、调语序、改连接词
- ✅ **词级替换优先于口语标记**：先把每个词换成更像人说的
- ✅ **格式零改动**：胶囊小标题不动、图片顺序不动、HTML 骨架不动
- ✅ 保留原文信息密度和逻辑连贯性

#### 禁区

- ❌ 不自行切换为群聊体/午夜好友体
- ❌ 不大改文章结构
- ❌ 不编造虚构场景来降 AI
- ❌ 不破坏 HTML 排版结构（胶囊小标题、图片顺序、模板结构）
- ❌ 不用口语标记灌爆文章（"你想想""讲真""说真的"全文 ≤ 2 个）

---

### 高频 AI 词替换速查

以下是最容易触发 AI 检测的词，优先替换：

**逻辑连接词：**
| 高频 AI 词 | 替换为 |
|:--------|:-----|
| 值得注意的是 | 注意 / 特别说一下 |
| 综上所述 | 说到底 / 简单讲 |
| 不难发现 | 可以看到 / 很明显 |
| 与此同时 | 同时 / 这时候 |
| 因此 | 因而 / 为此 |
| 然而 | 但 / 可是 |

**动词/行为词：**
| 高频 AI 词 | 替换为 |
|:--------|:-----|
| 赋能 | 帮助 / 提升 |
| 助力 | 帮助 / 支持 |
| 闭环 | 完整流程 |
| 进行 | 做 / 搞 |
| 实现 | 达成 / 做到 |
| 标志着 | （直接删除或改事实陈述） |
| 意味着 | （直接删除或改事实陈述） |

**替换纪律：**
- 每 1000 字至少替换 3-5 处
- 同一替换全文不超过 2 次
- 替换后通读确认句子通顺

---

### AI 骨架速查

当词级替换两轮后仍然僵硬时，检查是否命中以下骨架：
- 背景→事件→原因→影响→结论（线性分析报告结构）
- "标志着""这是一次升级""方向已经清楚了"（过度强调意义）
- "真正的关键是""本质上是"（深层揭秘腔）
- 三段式列举（不是…而是…/一方面…另一方面…）

**命中了 → 打断结构，不要只换词。**

---

### 格式保护自检

降 AI 后逐项检查：
- [ ] 胶囊式小标题（`<div class="titlestyle"><span>...</span></div>`）还在？
- [ ] 胶囊 CSS 完整？（`padding:6px 24px; border-radius:50px;`）
- [ ] HTML 结构仍然是：标题→摘要→首段→（小标题→正文）×N→结尾？
- [ ] 每节图片位置未变？
- [ ] verify 脚本仍能通过？

---

### 段落密度

如果原文段落过密（每段 2-3 句堆在一起），打碎成一句一段为主、偶尔两句合并。每段不超过 3 句。
