# 发布员

发布员是可选扩展，负责把审校归档后的公众号 HTML 稿件创建到平台草稿箱。

## 它解决什么问题

写作流程完成后，用户可能还要手动复制 HTML、上传图片、创建草稿。发布员把这一步变成可选自动化，但只做到草稿箱，不做最终群发。

## 它负责什么

- 读取审校归档后的 HTML
- 检查发布接入配置是否完整
- 按平台能力创建草稿
- 保留人工最终确认
- 在配置缺失时回到账号配置员

## 它不负责什么

- 不最终群发
- 不替用户点击正式发布
- 不读取聊天里的密钥
- 不绕过平台权限、认证和网络限制
- 不把 AppSecret 或 access_token 写进公开仓库

## 前置条件

- 审校已经完成归档
- 账号配置员已经引导完成 AppID、AppSecret 和网络允许列表配置
- 用户明确表示需要创建草稿箱草稿

## 推荐安装

发布员依赖总包里的账号配置、归档稿件和安全边界，不建议脱离总包单独安装。默认建议先关闭发布员，只跑写作闭环。

```bash
npx skills add https://github.com/Shinchan-crayon/wechat-writing-employee-pack/tree/main/skills/wechat-writing-employee-pack
```

安装后对 AI 助手说：

```text
请读取公众号写作员工包。我已经完成审校归档，并明确需要启用发布员创建公众号草稿。
```
