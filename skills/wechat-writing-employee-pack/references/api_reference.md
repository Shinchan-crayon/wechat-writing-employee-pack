# 工具调用示例

这个文件不是后端 API 文档，而是给用户和 AI 助手看的「公众号写作员工包调用示例」。

公众号写作员工包推荐按完整流程运行，不建议直接跳到写手。它的核心价值不是让 AI 立刻写一篇文章，而是先把账号配置、素材整理、事实核查、正文写作和审校归档拆开，降低事实错误和 AI 味。

## 推荐启动方式

安装总包：

```bash
npx skills add https://github.com/Shinchan-crayon/wechat-writing-employee-pack/tree/main/skills/wechat-writing-employee-pack
```

第一次使用时，对 AI 助手说：

```text
请读取公众号写作员工包，先执行账号配置员。不要直接跳到写手。
```

账号配置完成后，再按下面顺序执行：

```text
账号配置员 -> 素材捕手 -> 事实核查员 -> 写手 -> 审校 -> 发布员（可选）
```

## 常见调用场景

### 1. 第一次配置账号和风格

适合首次安装、换账号、换写作风格时使用。

```text
请执行公众号写作员工包的账号配置员。

帮我创建本地 USER.md，确认公众号账号信息、文章类型、目标读者、常用表达风格、禁用词和工作区路径。

我现在先不启用发布员，只跑写作和审校流程。
```

预期结果：

- 生成或更新本地 `USER.md`
- 明确文章类型、读者画像和风格卡
- 确认是否启用发布员
- 不写入真实密钥到公开仓库

### 2. 根据一个选题写文章

适合你只有一个方向，还没有整理完整素材时使用。

```text
请按公众号写作员工包流程处理这个选题：

选题：国产大模型价格战之后，AI 应用创业公司的成本结构会怎么变？

要求：
1. 先让素材捕手整理背景资料和文章框架
2. 再让事实核查员核查价格、时间线和关键案例
3. 核查通过后再进入写手
4. 最后由审校做降 AI 味、日期核查和归档

不要直接生成最终全文。
```

预期结果：

- 素材清单
- 文章框架
- 事实核查表
- HTML 正文草稿
- 标题候选
- 审校记录

### 3. 根据已有素材改写成公众号文章

适合你已经有链接、访谈稿、会议纪要或资料包时使用。

```text
请按公众号写作员工包流程，把下面素材整理成一篇公众号文章。

素材：
- 链接 1：<粘贴链接>
- 链接 2：<粘贴链接>
- 我的观点：<粘贴你的观点>

请先让素材捕手提取素材要点、补充必要背景，再由事实核查员检查关键数据和引用。

文章风格沿用 USER.md 里的账号风格，不要写成通稿。
```

预期结果：

- 原始素材摘要
- 可引用事实列表
- 文章主线和分段结构
- 可复制到公众号编辑器的 HTML 稿件

### 4. 只做事实核查

适合你已经写好了草稿，但担心数据、案例、时间线出错。

```text
请只调用事实核查员，检查下面这篇公众号草稿。

重点核查：
1. 数据和价格是否有来源
2. 时间线是否准确
3. 公司、产品、人物名称是否写错
4. 是否存在过度断言
5. 哪些句子需要补来源或改成更稳妥表达

草稿如下：
<粘贴草稿>
```

预期结果：

- 高风险事实清单
- 需要补来源的句子
- 建议删除或改写的表述
- 可保留但需要降调的判断

### 5. 只做审校和降 AI 味

适合草稿已经完成，只需要更像公众号作者本人。

```text
请调用审校，对下面这篇文章做最终审校。

要求：
1. 降低 AI 味
2. 检查开头是否太模板化
3. 检查小标题是否太像生成式文本
4. 检查日期、数据、引用和风险表达
5. 输出审校后的 HTML 稿件和审校记录

草稿如下：
<粘贴草稿>
```

预期结果：

- 审校后正文
- 问题修改记录
- 日期和事实核查提醒
- 最终归档建议

### 6. 创建公众号草稿箱草稿

发布员是可选能力，只在你已经完成配置，并明确要求时使用。

```text
请调用发布员，把已归档的 HTML 稿件推送到公众号草稿箱。

注意：
1. 只创建草稿
2. 不要最终发布
3. 推送前先检查 USER.md 和本地 .env 配置是否完整
4. 推送失败时说明错误原因，不要重复盲试
```

预期结果：

- 创建草稿箱草稿
- 返回草稿创建结果
- 不执行群发或最终发布

## 执行边界

- 总包是推荐安装方式，子 Skill 不建议脱离总包单独运行。
- `USER.md`、`.env`、真实 AppSecret、真实 access_token 只应保留在用户本地。
- 发布员默认关闭，只创建草稿，不做最终发布。
- 涉及数据、价格、融资、排名、案例和引用时，应先核查再写正文。
- 用户明确要求快速草稿时，也应至少保留事实风险提示。
