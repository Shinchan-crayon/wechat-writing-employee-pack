# 写手

写手负责基于已经核查过的框架和素材，生成公众号 HTML 正文和标题候选。

## 它解决什么问题

写手不是从零开始编文章，而是在账号风格、文章框架和事实核查结果都明确之后，再进入正文生成。这样可以减少 AI 味、减少瞎编，也更容易保持公众号账号的一贯表达。

## 它负责什么

- 按已确认框架写公众号正文
- 使用账号风格卡控制语气和表达
- 输出适合公众号编辑器的 HTML
- 生成多组标题候选
- 保留关键事实和来源约束

## 它不负责什么

- 不跳过事实核查直接写作
- 不伪造数据、引用或来源
- 不负责最终审校
- 不负责创建平台草稿

## 前置条件

- 事实核查员已经确认关键数据和来源
- 用户确认文章框架可以进入写作
- 账号风格卡已经配置

## 推荐安装

写手依赖总包里的模板、风格卡、核查结果和后续审校流程，不建议脱离总包单独安装。

```bash
npx skills add https://github.com/Shinchan-crayon/wechat-writing-employee-pack/tree/main/skills/wechat-writing-employee-pack
```

安装后对 AI 助手说：

```text
请读取公众号写作员工包，基于已核查框架启动写手，输出公众号 HTML 正文和标题候选。
```
