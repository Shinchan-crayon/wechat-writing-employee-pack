# 配置网络权限

## 为什么需要

平台 API 通常要求调用方的网络地址在允许列表内。不在列表 = 所有 API 请求返回权限拒绝错误。

你可能需要配置多台机器的网络地址：
- **本机地址**（开发时调试用）
- **服务器地址**（如果部署在云端）

## 第一步：查本机网络出口地址

优先让账号配置员运行包内脚本查询本机公网出口 IP：

```bash
python3 ${PACKAGE_ROOT}/脚本/check-outbound-ip.py
```

如果运行时不允许联网，也可以手动打开任意可信 IP 查询网站，查看当前公网出口 IP。

⚠️ 如果你的网络环境是家用宽带，地址可能会在重启路由器后变化。变化后需要重新添加到允许列表。

## 第二步：添加到平台允许列表

1. 登录平台开发者后台
2. 进入「基本配置」页面
3. 找到「IP 允许列表 / 安全配置」区域
4. 添加第一步查到的地址

如果有多台机器（如本机 + 云服务器），每行一个地址。

## 第三步：确认配置

在 `USER.md` 中填写：

```yaml
publisher:
  platform:
    request_ip: "查询到的公网出口 IP"
    network_configured: true
```

## 通过标准

- 平台后台允许列表中可以看到你的地址
- `USER.md` 中 `publisher.platform.network_configured` 为 `true`

## 常见问题

**Q：我的地址会变吗？**
A：家用宽带地址通常是动态的，重启路由器可能导致变化。解决办法：
- 每次地址变化后重新添加
- 或者使用固定地址的云服务器做中转
