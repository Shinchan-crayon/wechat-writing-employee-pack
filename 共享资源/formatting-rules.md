# 排版规则

所有排版相关约束集中于此。写手和审校员工均需遵守。

## HTML 骨架要求

**必须从模板开始改，不从零手写。** 模板路径由各员工的 policy.yaml 指定。

## 正文排版

| 属性 | 值 |
|------|-----|
| 标题字号 | 18px，居中，`color:#333`，margin/padding=0 |
| 正文字号 | 17px，`color:#3f3f3f` |
| 行距 | 1.6 |
| 字间距 | 0.05em |
| 段落下间距 | 1.5em |

## 小标题（胶囊式）

```css
.titlestyle {
  text-align: center;
  margin: 24px 0;
}
.titlestyle span {
  color: #ffffff;
  background: #8B2E2E;
  padding: 6px 24px;
  border-radius: 50px;
  font-size: 18px;
  font-weight: bold;
}
```

### ⚠️ 胶囊 CSS 完整性检查

多次重写 HTML 时容易丢失 `.titlestyle span` 中的 `padding:6px 24px; border-radius:50px;`。交付前必须确认这两项存在。缺少 = 红色胶囊不可见。

## 小标题格式

统一使用「」胶囊式，例如：
- 「这玩意儿离谱在哪」
- 「对手盘这波直接就吃麻了」
- 「Sora为什么撑不下去呢？」

禁止编号目录式：`1. 事件背景` / `一、核心原因` / `首先 / 其次 / 最后`

## 关键词标红

```css
.hl {
  color: #a93226;
  font-weight: bold;
}
```

标红对象：核心数字、关键判断、产品名/模型名首次出现时。每段最多 1-2 处，全篇不超过 15 处。

## 段落规则

- 每段 1-2 句，绝不出现 3 句段
- 关联的 2 句合并，单独的重点句/情绪句独立成段
- 陈述句段尾不加句号，问句保留问号，感叹句保留感叹号
- 只在大的逻辑块之间空行（小节切换、情绪转折），同一小节内段落不空行
- 每段开头优先直接给事实，不先铺情绪判断

## 禁止的排版

- 每句都独立成 `<p>`（段落太碎，像诗）
- 连续 3 段以上单句段落（在非风格化场景下）
- 一段塞 3 句以上（像论文）
- 胶囊式小标题混用编号格式
- `<h1>/<h2>/<h3>` 有额外 margin-top/margin-bottom

## 默认无配图

所有文章默认不放配图。HTML 中不插入 `<img>` 标签或 `<div class="img-wrap">`。仅当用户明确要求配图时才走配图流程。
