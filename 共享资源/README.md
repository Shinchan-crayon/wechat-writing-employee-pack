# 写作风格卡 — 使用说明

## 这是什么

`style.md` 是本员工包提供的一份**参考模板**。它定义了一套写作风格：标题多长、开头怎么写、什么词不能用、段落节奏怎么控。

**它不是唯一风格。** 你可以（也应该）改成自己的。

## 三种用法

### 1. 直接用（改都不改）

如果你的公众号调性刚好匹配这份风格（犀利、短句、有网感、面向科技/财经读者），可以直接用。把 `style` 字段指向本文件即可。

### 2. 基于模板改

```bash
cp 共享资源/style.md 共享资源/my-style.md
# 编辑 my-style.md，改标题字数、语气、新增你的禁用词
# 在本地 USER.md 的 accounts.style 中指向 共享资源/my-style.md
```

### 3. 从零写

按 `style-schema.md` 的字段定义，写一份全新的风格卡。字段名和结构保持一致，adapter 才能正确解析。

## 风格卡的工作原理

- 每个员工在启动时读取本地 USER.md 中 accounts.style 指定的风格卡
- 风格卡是**软约束**——它定义「怎么写」，不定义「能不能做」
- 「能不能做」（访问哪些文件、能否执行脚本等）由 `policy.yaml` 控制
- 同一套员工包可以配不同的风格卡，适配不同公众号

## 如果你觉得默认风格不够好

这份模板来自一个真实公众号的实战沉淀，但它有自己的偏好（偏犀利、偏吐槽风）。如果你想要更温和、更学术、更活泼的风格，改 `style.md` 里的对应字段就行——不需要改任何员工的 IDENTITY 或 SOUL。
