# ThinkAI Skill 广场上架文案

本仓库采用“一个仓库，多入口上架”：1 个总包入口 + 6 个员工子入口。正式推荐用户安装总包；子入口用于广场展示、说明职责和帮助用户理解流程。

## 统一安装命令

```bash
npx skills add https://github.com/Shinchan-crayon/wechat-writing-employee-pack
```

## 版本记录

v1.0.0：首个公开版本。包含五个核心员工、一个可选发布员、共享风格卡、HTML 模板、事实核查资料、标题生成资料、降 AI 资料、校验脚本、私有配置模板和首次运行自检文档。

## 总包入口

**卡片名称：** 公众号写作员工包

**一句话简介：** 把公众号写作拆成账号配置、素材搜集、事实核查、写稿、审校和可选草稿推送的 AI 员工包。

**详情描述：** 这是一套公众号写作 Workflow 总包，适合中文科技、财经、行业分析类公众号。流程为：账号配置员 -> 素材捕手 -> 事实核查员 -> 写手 -> 审校 -> 发布员。首次运行先生成本地 USER.md，发布员默认可选，配置完成前只跑写作闭环。

**首条使用提示：**

```text
请读取公众号写作员工包，先执行账号配置员。不要直接跳到写手。按「账号配置员 -> 素材捕手 -> 事实核查员 -> 写手 -> 审校」顺序完成文章；发布员默认跳过，等核心写作闭环跑通后再配置。
```

**GitHub 资源路径：** `README.md`, `manifest.yaml`, `首次运行自检与故障反馈.md`, `共享资源/`, `模板/`, `脚本/`, `参考资料/`, `私有配置模板/`

## 子入口：账号配置员

**卡片名称：** 账号配置员

**一句话简介：** 初始化公众号账号、风格卡、本地私有配置和可选发布接入。

**详情描述：** 账号配置员是公众号写作员工包的第一步。它负责引导用户生成本地 USER.md，确认公众号账号、工作区、风格卡，并在用户需要时引导配置发布员所需的 AppID、AppSecret 和网络允许列表。它不写文章，不读取未授权密钥，不把凭证写入公开包。

**首条使用提示：**

```text
请先执行账号配置员，基于私有配置模板生成本地 USER.md，确认账号、工作区和风格卡；如果暂时不需要发布能力，请跳过发布员，只进入写作闭环。
```

**GitHub 资源路径：** `00-账号配置员/`

## 子入口：素材捕手

**卡片名称：** 素材捕手

**一句话简介：** 读取账号风格和源素材，搜索补充信息，并搭建公众号文章框架。

**详情描述：** 素材捕手负责工作流第 1-4 步：读取本地账号配置和风格卡，理解用户给定的源素材，补充搜索 3-5 个可靠来源，并输出可确认的文章框架。它不直接写完整文章，不跳过用户确认，也不替代事实核查员做最终核查。

**首条使用提示：**

```text
请作为素材捕手读取账号配置、风格卡和源素材，补充搜索可靠来源，然后先给出文章框架供我确认，不要直接写正文。
```

**GitHub 资源路径：** `01-素材捕手/`, `共享资源/`, `参考资料/事实核查/`

## 子入口：事实核查员

**卡片名称：** 事实核查员

**一句话简介：** 对数据、价格、案例和引用进行写前建表核查与写后逐句反查。

**详情描述：** 事实核查员负责工作流第 5 步和第 8 步。凡是涉及价格、融资、参数、排名、监管、案例和关键引用，都需要建立核查表并尽量追到官方或一手来源。成稿后还要逐句反查，避免把二手转述当事实。

**首条使用提示：**

```text
请作为事实核查员，对素材捕手给出的框架和数据建立核查表；所有关键事实必须标注来源和可信度，未核实项不要放行给写手。
```

**GitHub 资源路径：** `02-事实核查员/`, `参考资料/事实核查/`

## 子入口：写手

**卡片名称：** 写手

**一句话简介：** 基于已核查框架生成公众号 HTML 正文和标题候选。

**详情描述：** 写手负责工作流第 6-7 步。它接收事实核查员放行的框架和数据，按包内 HTML 模板生成公众号正文，并给出标题候选。写手不跳过事实核查，不伪造来源或数据，也不负责最终审校和归档。

**首条使用提示：**

```text
请作为写手，只基于已经核查通过的框架和数据写公众号 HTML 正文，并给出标题候选；不要新增未经核查的事实。
```

**GitHub 资源路径：** `03-写手/`, `模板/article.html`, `参考资料/标题生成/`

## 子入口：审校

**卡片名称：** 审校

**一句话简介：** 对成稿进行降 AI 味、9 项自检、日期核实和归档。

**详情描述：** 审校负责工作流第 9-11 步。它对写手交付的 HTML 成稿进行降 AI 味处理、9 项自检、日期核实、风险表达检查和归档。审校不重写整篇结构，不放过未核实日期，也不负责平台发布。

**首条使用提示：**

```text
请作为审校检查这篇公众号 HTML 成稿，重点处理 AI 味、事实表达、日期、排版、标题和风险表述；通过后再归档。
```

**GitHub 资源路径：** `04-审校/`, `共享资源/`, `参考资料/降AI/`

## 子入口：发布员

**卡片名称：** 发布员

**一句话简介：** 可选扩展，把审校归档的 HTML 推送到平台草稿箱，不最终发布。

**详情描述：** 发布员是可选扩展，只在账号配置员已完成 AppID、AppSecret 和网络允许列表配置后启用。它负责把审校归档的 HTML 创建为平台草稿，保留人工最终确认。它不最终群发，不读取聊天中的密钥，也不绕过平台权限和网络限制。

**首条使用提示：**

```text
请作为发布员先检查 USER.md 中发布配置是否完成；如果未完成，请回到账号配置员。配置完成后，只把归档 HTML 推送到草稿箱，不执行最终发布。
```

**GitHub 资源路径：** `05-发布员/`, `私有配置模板/发布员配置引导/`, `脚本/check-outbound-ip.py`
